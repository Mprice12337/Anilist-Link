How to get your AniList credentials
  1. Go to anilist.co/settings/developer and sign in.
  2. Click Create New Client.
  3. Set the Redirect URI to http://THIS_SERVER/auth/anilist/callback.
  4. Copy the Client ID and Client Secret into the fields below.
  5. Click Save & Test, then Authorize AniList to link your account.

How to find your Plex server URL & token
Server URL — your Plex server's local address, e.g. http://192.168.1.100:32400. Find it in Plex Web → Settings → Remote Access, or check your router's DHCP list for the host running Plex.

Admin Token — a unique key that identifies your Plex account to the server.
  1. Open app.plex.tv in your browser and sign in.
  2. Open browser DevTools (F12) → Network tab.
  3. Play or click any media item. Look for a request URL containing X-Plex-Token= — that value is your token.
  4. Tip: that same request URL contains your server address. You can paste the full URL into the Server URL field above and it will automatically be trimmed to just the base (e.g. https://192-168-x-x…plex.direct:32400).
Alternatively, see the Plex support article for other methods.

How to find your Jellyfin server URL & API key
Server URL — your Jellyfin server's local address, e.g. http://192.168.1.100:8096. You can find or confirm this in Jellyfin → Dashboard → Networking.

API Key — generated in the Jellyfin admin panel.
  1. Open your Jellyfin server and sign in as an administrator.
  2. Go to Dashboard (top-right menu) → Advanced → API Keys.
  3. Click the + button, give it a name (e.g. Anilist-Link), and click OK.
  4. Copy the generated key into the field below.
Webhook URL
How to add this in Jellyfin:
Go to Dashboard → Plugins → Catalogue, install the Webhooks plugin if not already installed, then restart Jellyfin.
After restarting, go to Dashboard → Plugins → Webhooks and click Add Generic Destination. Configure it as follows:

• Webhook Name — anything (e.g. Anilist-Link)
• Webhook URL — paste the URL above
• Notification Types — enable Playback Stop and User Data Saved. Playback Stop fires when an episode ends (includes whether it played to completion); User Data Saved catches manual mark-as-watched actions. Playback Progress fires every ~10 seconds and is not needed.
• User Filter — leave empty to sync all users, or select specific users whose watch history you want to track
• Item Types — enable Episodes and Movies (at least one must be checked or no events will fire)
• Send All Properties — check this box so the full playback data (episode number, completion status, position, etc.) is included in the payload
• Headers — add Content-Type: application/json
• Server URL — optionally set to your Jellyfin address (); only affects media image URLs in the payload, not webhook delivery

About Crunchyroll setup
Enter your Crunchyroll email and password — the same credentials you use to log in on crunchyroll.com.

FlareSolverr URL (optional) — Crunchyroll uses Cloudflare protection that can block automated requests. FlareSolverr is a small proxy service that bypasses this. If you don't have it running, leave this field blank and Anilist-Link will attempt a direct connection. To run FlareSolverr with Docker:
  1. Add flaresolverr/flaresolverr as a container (port 8191).
  2. Enter http://YOUR-HOST:8191 in the field below.
Headless mode — runs the browser invisibly in the background. Leave this on unless you need to debug the login process.
